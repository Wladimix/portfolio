Приложение курьерской службы доставки.

Веб-приложение представляет собой сервис для управления заказами и курьерами. Позволяет заносить данные об организации в систему, после чего можно работать с её заказами. Многие организации-клиенты (преимущественно рестораны) интегрированы с сервисом управления организацией в облаке. Структура приложения - монолит. Серверная и клиентская части располагаются в одной директории. Веб-сервер - Nginx.

Задачи и обязанности:

  • поддержка и доработка приложения (ревью кода, поиск и исправление багов, добавление нового функционала и обновление существующего)
  • интеграция с внешними сервисами, преимущественно с IikoCloud (настройка конечных точек для работы с IikoCloud API, настройка шлюзов)
  • управление базой данных (работа с очередями задач, расширение посредством миграций, настройка фабрик и сидеров при добавлении нового функционала)
  • доработка клиентской части (вёрстка и добавление новых страниц с логикой обработки и отображения данных, полученных с сервера)
  • обновление функционала в мобильном приложении курьера
  • иногда приходилось настраивать конфигурацию Nginx
HTML, CSS PHP 7 (Laravel, CRUDBooster) JavaScript (JQuery, React, Effector) Kotlin SQL (MySQL) Webpack

Клиентское приложение для управления спортивным табло

Приложение для управления спортивным табло в спортзале (как с планшета, так и с настольного ПК). Предназначается для отслеживания текущей игры. Управляет раундами (таймами), определяет команду-победителя (команду-проигравшего, или ничью) согласно правилам текущей игры (футбол, баскетбол, настольный теннис и другие), отслеживает текущие счёт и время игры, управляет бегущей строкой на табло, запускает демосцены между играми, или во время игр, отображает информацию о командах и о следующей игре, если она состоится.

Задачи и обязанности:

Разработка приложения с нуля на основе предоставленного API серверной части:

  • работа с модулем управления демосценами (вёрстка и разработка логики отображения и запуска доступных демосцен)
  • работа с модулем управления счётом и настроек текущей и предстоящей игры (вёрстка и разработка логики отображения информации об играх и управления текущей игрой)
  • работа с модулем управления бегущей строкой (вёрстка и разработка логики редактирования и запуска бегущей строки, а также шаблонов текста бегущей строки)
HTML, CSS (Bootstrap) JavaScript (React, Redux) Webpack

Серверная часть сайта кадрового центра Москвы

Веб-приложение для размещения вакансий кадрового центра Москвы. Пользователь может зарегистрироваться стандартным способом, а может откликнуться на вакансию в качестве гостя с автоматическим созданием аккаунта и отправкой информации (логин и пароль) на почту. Каждый аккаунт синхронизируется с дистанционной системой обучения Мираполис. Менеджеры могут управлять вакансиями, пользователи - откликаться, администраторы - и то, и другое. Приложение имеет веб-версию (сайт) и терминальную версию (сенсорные терминалы в кадровом центре). Каждая вакансия имеет фото и видео-обложку.

Задачи и обязанности:

Разработка серверной части приложения с нуля и дальнейшая поддержка:

  • проектирование архитектуры под шаблон MVC
  • настройка основных CRUD-операций (пользователи, вакансии, медиа) и описание API в Swagger
  • разработка логики загрузки статических файлов (фото и видео к вакансиям)
  • проектирование базы данных
  • работа с авторизацией (использование API-токена)
  • создание системы ролей (пользователь, менеджер вакансий, мл. администратор, администратор)
  • разработка логики интеграции с дистанционной системой обучения Мираполис (шлюзы, очереди задач)
  • работа с отправкой данных аккаунта на почту зарегистрированного пользователя (настройка SMTP-сервера, формирование QR-кода с id пользователя, вёрстка шаблонов писем)
  • поддержка и доработка приложения (ревью кода, поиск и исправление багов, добавление нового функционала и обновление существующего).

Работа в ЭМ СИ АРТ

Здесь было сразу несколько проектов на Bitrix (в основном - модули для официального маркетплейса). Наиболее крупный из них - это тиражное публичное решение для корпоративного портала Bitrix24. Этот функционал представляет собой приложение, написанное на Nest.JS (сервер) и Vue.JS (клиент), интегрированное в Bitrix24. Я работал с серверной частью. Поэтому необходимо было работать также с API системы Bitrix24 и вебхуками. Приложение представляло собой функционал для общения с операторами службы поддержки Avito через мессенджеры Telegram и WhatsApp.
